# 诊断卡模板

当对单个技能进行审查时，使用以下卡片格式输出。

## 基本格式

```markdown
## {{技能名称}} 诊断卡

**类别**: {{大类/小类}}
**平台**: {{Hermes | Claude Code | Codex}}

### 路由健康

| 检查项 | 状态 | 详情 |
|--------|------|------|
| 模板残余 | ✅/❌ | 具体描述 |
| 中英混合 | ✅/❌ | 具体描述 |
| 弱负样本 | ✅/❌ | 具体描述 |
| tags | ✅/❌ | 具体描述 |
| related_skills | ✅/❌ | 具体描述 |

### 修改建议

```yaml
# 修改后的描述
description: |
  【触发场景】当{{触发场景}}时
  【输入特征】涉及"{{关键词1}}", "{{关键词2}}", "{{关键词3}}"等表述
  【输出目标】{{输出描述}}
  【不适用场景】{{负样本1}}；{{负样本2}}
```

### 严重度

- 🔴 必须处理: 具体项目
- 🟡 建议处理: 具体项目
- 🟢 可选优化: 具体项目
```

## 批量输出格式

当同时审查多个技能时，按以下格式分组输出：

```
=== 技能批量诊断（10个）===

### 1. {{技能名}} [🔴/🟡/🟢]
严重: {{具体项目}}
建议: {{具体项目}}

### 2. {{技能名}} [🔴/🟡/🟢]
严重: {{具体项目}}
...
```
